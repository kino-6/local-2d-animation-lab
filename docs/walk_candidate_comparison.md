## Current Interpretation

- The best route remains staged single-keyframe Wan i2v for base motion, followed by BiRefNet separation and strict artifact/review labeling.
- VACE contact-swing remains useful as a controlled-motion comparator, but it is no longer the preferred subject-preservation route.
- Visible foot-guide/control-map overlays are not the current mainline. They are rejected diagnostics until a workflow can consume foot/contact constraints without rendering the guide pixels into the output.
- The latest blocker is not foreground shrinkage; it is recurring pale lower-body afterimages inside otherwise readable walking frames.
- The latest local reproduction confirms the route is still viable, but the current contact-pose start frame is weaker than the missing historical phase10 start image: it begins with `shoes_unreadable`, and the generated clips keep lower-body/foot labels.
- The next retake should improve the start frame or source crop before spending on full 120-frame generation, then tune single-keyframe Wan i2v prompts/settings and post-generation polish to reduce ghosted lower-body motion.

## Improvement Boundary

Postprocess can help with:

- global luma or saturation jitter;
- beige/dirty background cleanup;
- small specks or minor residual ghosts;
- 1024 img2img polish after action semantics and identity are already correct.

Retake or route change is required for:

- double feet or duplicated lower legs;
- guide/control-map pixels visible in the generated frame;
- character identity loss or silhouette collapse;
- major lower-body redraw across many frames;
- motion that fails to read as a walk.

The practical next step is not to add stronger visible control. It is to stay on the single-keyframe Wan i2v route, recover or regenerate a stronger walk-ready start frame than `outputs/20260613_174107/.../selected_reference/start_frame.png`, then repeat the 768 short probe before any 120-frame spend.
